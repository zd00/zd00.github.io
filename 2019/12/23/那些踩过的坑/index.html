¶Git Bash

¶Git提交代码时由于说明含中文而报错

$ Warning: Your console font probably doesn't support Unicode. If you experience strange characters in the output, consider switching to a TrueType font such as Consolas!

  • 解决办法：设置命令端字体属性

¶warning: LF will be replaced by CRLF in

报错原因是windows提交的内容存在符号（换行符）转义的问题，不同操作系统所使用的换行符是不一样的：Dos/Windows是CRLF、Unix/Linux是LF、Mac OS是CR
解决办法：配置git转义的方式
$ git config --global core.autocrlf false

为true时，Git会将你add的所有文件视为文本，将结尾的CRLF转换为LF，而checkout时会再将文件的LF格式转为CRLF格式。（推荐此设置，多人跨平台开发环境）

为false时，line endings（行尾）不做任何改变，文本文件保持其原来的样子。（为单独windows开发环境设计）

为input时，add时Git会把CRLF转换为LF，而check时仍旧为LF，所以Windows操作系统不建议设置此值。
参考资料来源：CSDN站点/Starry_Night9280
