Github Pages设置

GitHub Pages 作用是介绍托管在GitHub的项目，由于空间免费稳定，所以常被用来做搭建个人网站。

  1. 注册github账号，新建仓库并命名为 username.github.io

注意：（前缀username 是你注册的账号ID）
点击文字链接可进入相应页面

  1. 安装Git（用于本地提交代码到github仓库）^1
    Windows下载 默认安装就行 其他系统 [^2]

ElvisCT(CSDN)《Git安装教程 (windows)》

[^2]:Scott Chacon、Ben Straub《Pro Git》

打开GitBash来到命令行终端，此时可编辑网页展示文件test.md推送到github进行简单测试：（可跳过这步）

  • 新建文件夹blog并初始化作为Git代码库
    $ git init blog
  • 在工作区blog创建编辑展示的md文件并提交到暂存区
  • 将暂存区提交到版本库并连接到远程仓库进行推送部署
  • 关于git更多信息 感兴趣的可以看看 ^3

ok 现在你可以通过访问userid.github.io访问创建的展示页面了！

Hexo搭建

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。^[hexo中文开发文档]

环境配置：

  • Node.js官方下载
  • 参考安装 ^[站点runoob.com/《 Node.js 安装配置》]

安装Hexo NexT.Pisces v7.6.0

  • 打开GitBash终端，若应用在D盘
    此时你在 '/d/hexo/blog/' 发现生成了以下结构文件： 
    
├── node_modules       //依赖安装目录
├── scaffolds          //模板文件夹，新建的文章将会从此目录下的文件中继承格式
|   ├── draft.md         //草稿模板
|   ├── page.md          //页面模板
|   └── post.md          //文章模板
├── source             //资源文件夹，用于放置图片、数据、文章等资源
|   └── _posts           //文章目录
├── themes             //主题文件夹
|   └── landscape        //默认主题
├── .gitignore         //指定不纳入git版本控制的文件
├── _config.yml        //站点配置文件
├── db.json            
├── package.json
└── package-lock.json

    Hexo主题安装配置过程

  • 下载主题到themes目录下，以next主题为例：
    $ git clone https://github.com/theme-next/hexo-theme-next.git themes/next

  • 启用主题

    打开根目录下`_config.yml`文件,将`theme`字段的值修改为`next` 
    
# Extensions
## Plugins: https://hexo.io/plugins/
## Themes: https://hexo.io/themes/
theme: next
